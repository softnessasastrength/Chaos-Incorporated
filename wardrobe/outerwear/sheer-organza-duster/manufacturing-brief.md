# The Sheer Organza Duster — Manufacturing Brief

**Document Type:** Technical Specification / Manufacturing Brief for pattern development and factory quoting
**Prepared for:** Garment manufacturer producing lightweight sheer outerwear (organza/chiffon)
**Revision:** 1.0 (Draft — pending manufacturer confirmation of crop-length pattern and sleeve construction)
**Governing doc:** [`vision.md`](vision.md) — read for design rationale; this document is the buildable spec.

---

## 1. Product Overview

An open or single-clasp duster in fully translucent organza or silk/poly chiffon, 34–38" length, 3/4 or pushable sleeve, no internal structure or opacity of its own. Four colorways share one pattern.

## 2. Pattern & Construction Specification

| Parameter | Target | Notes |
|---|---|---|
| Length | 34–38" | **Reject floor-length samples** — confirm the manufacturer's base pattern is proportioned for this crop, same verification approach as the Kimono Duster, not just hemmed down from a longer duster block |
| Sleeve | 3/4 or pushable to clear the wrist | **Gating spec** — confirm against a live fit model in motion; full sheer sleeves are the highest hand-visibility risk on this piece |
| Closure | Open, or single silver-tone clasp / ribbon tie | |

## 3. Fabric Specification

| Parameter | Target | Notes |
|---|---|---|
| Fabric | Soft organza or silk/poly chiffon | 30–70 gsm |
| Hand-feel | Barely-there, no stiff crackle | Confirm against physical swatch — lower-quality organza can read as stiff or noisy rather than soft and quiet |
| Beading/trim | None | No heavy beading anywhere on this piece — confirm no default trim additions from a standard duster template |

## 4. Opacity Note (Not a Manufacturing Spec, but Worth Flagging)

Unlike the Sheer Overlay Dress, this garment has no attached underlayer and is not expected to independently pass any opacity test — the entire piece is translucent by design, with opacity coming from whatever's worn underneath. Confirm this is understood by the manufacturer so no lining or opacity treatment gets added by default; a manufacturer accustomed to sheer-garment opacity concerns may reflexively suggest a lining that isn't wanted here.

## 5. Colorway Spec Summary

| Colorway | Role |
|---|---|
| Cloud Ivory | Core |
| Starlink Emerald | Hero veil colorway — confirm sheer chiffon/organza color match against `../../color-system.md`'s `#0F5C3D` reference; sheer fabrics can read differently in color than opaque fabrics at the same dye formula |
| Blush Mauve | Core |
| Lavender Mist | Core |

**Recommended build order:** Cloud Ivory first (simplest color, lowest risk), then Starlink Emerald once dye-match is confirmed.

## 6. Starlink One Reference Match

Silver-tone clasp only, if a clasp closure is used. No other hardware or debossed detail required.

## 7. Critical Technical Challenges — Open Items for Manufacturer Conversation

1. **True cropped-pattern proportions** — same verification as the Kimono Duster: confirm the pattern is genuinely built for this length, not hemmed down from a standard duster block.
2. **Sleeve-clears-wrist validation in motion** — the single highest-priority open item.
3. **No default opacity treatment** — confirm the manufacturer doesn't add lining or an opacity layer unprompted.
4. **Emerald sheer-fabric color match** — confirm against the color system reference, noting sheer fabric can render color differently than an opaque swatch at the same formula.

## 8. Requested Next Step

A single fit sample in Cloud Ivory, standard/reference size, evaluated for sleeve-clears-wrist behavior in motion, before any multi-colorway production commitment. Given this piece's Experimental tier, treat this as a design-validation sample rather than a near-term production commitment.
