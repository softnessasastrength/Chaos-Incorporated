**Every item in the catalogue now has the full Anchor One treatment** — a `vision.md` design-rationale document plus a Shenzhen-ready `manufacturing-brief.md`, in its own subfolder. The original single-file concept notes (right-hand column above) are left in place, unmodified in substance, since `full-looks/` and `bottoms/jeans/jeans-template.md` reference several of them directly by path — each now carries a one-line pointer to its fuller folder at the top.

**Naming note:** items 1–4 are renamed/upgraded versions of the original Phase 1–2 shoes, not new duplicate items — original filenames and paths stayed stable so existing cross-references (`full-looks/`, `bottoms/jeans/jeans-template.md`, etc.) don't break. The catalogue's "Statement Mule" concept mapped onto two real, distinct silhouettes already in the system (a closed pump and an open-back mule) rather than one — both are listed under #6 rather than picking one arbitrarily.

**Design principles applied across all eight:** low-to-mid rise / stable base / no aggressive pitch for a 64" frame; comfort non-negotiable (memory foam + real arch support); materials chosen to age like the ring, not resist aging; every pair carries at least one Starlink One reference, placed hidden on everyday pieces and visible on statement pieces — see each item's own Starlink One Pairing section for the specific rule being applied.

Note from the bottoms system: several silhouettes (wide-leg jean, flare jean, barrel jean) explicitly depend on a specific shoe category (heeled boot, platform sole) to complete their proportion logic — cross-reference `../bottoms/jeans/jeans-template.md` when sourcing so shoe purchases match the bottoms they're engineered to pair with.
